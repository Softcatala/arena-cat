# AGENTS.md

Guide for coding agents working in this repository.
For project context, see `README.md`, `projecte.md` and `pla_detallat.md`.

## Overview

Arena Cat is a platform for human evaluation of AI models in Catalan.

## Structure

- `backend/` — Python code: data model, migrations and tests. See `backend/README.md`.
- `infra/` — database initialization scripts.
- `docker-compose.yml` — local PostgreSQL.
- `simulador/` — sizing simulator (static web).
- `docs/` — project documentation (ER diagram, etc.).

## Getting started

```bash
cp .env.example .env
docker compose up -d --wait
cd backend && uv sync && uv run alembic upgrade head
uv run pre-commit install   # lint/format git hook (catches issues before CI)
```

## Common commands (from `backend/`)

- Tests: `uv run pytest` (`-v` to see each test name)
- Lint: `uv run ruff check .`
- Format: `uv run ruff format .`
- Migrations: see `backend/README.md`

## Conventions

- **Commits**: Conventional Commits, in English (`feat:`, `fix:`, `test:`, `docs:`, `chore:`).
- **Language**: identifiers in English (tables, columns, functions, variables, test names).
  Docstrings and inline comments in Catalan.
- **TDD**: write tests before the implementation whenever possible. Model tests run against
  a real PostgreSQL.
- **Comments**: describe the code, not the methodology. No process notes, no tool
  comparisons, no tying documentation to a specific issue (it must be able to grow with
  the project).
- **Environment**: managed with `uv` (no need to activate venv or pyenv). Do not commit `.env`.
- **Schema**: every database change goes through an Alembic migration. Always review the
  autogenerated files and keep the diagram in `docs/db_schema.md` in sync. While the schema
  is not deployed, regenerate the initial migration instead of stacking a second one.
- **Integrity**: prefer guarantees at the database level (foreign keys, uniqueness, CHECK)
  over application-level validation.
